# Guía de Contribución

Gracias por su interés en contribuir a este proyecto. Esta guía establece las expectativas para el desarrollo y la colaboración.

## Proceso de Desarrollo

1. Bifurque (Fork) el repositorio.
2. Cree una rama para su característica o corrección de errores desde la rama `develop` (o equivalente).
3. Escriba código limpio y documentado.
4. Asegúrese de que todas las pruebas pasen y escriba nuevas pruebas para sus cambios.
5. Envíe un Pull Request hacia la rama `develop`.

## Convenciones de Código

- Mantenga la consistencia con el estilo de código existente.
- Utilice linters y formateadores configurados en el proyecto.
- Evite la complejidad innecesaria y documente las decisiones de diseño.

## Pruebas

Todo código nuevo debe estar respaldado por pruebas unitarias o de integración. Las pruebas deben ejecutarse localmente antes de enviar un Pull Request para asegurar que no se rompan funcionalidades existentes.

## Revisiones de Código

Todos los Pull Requests requieren al menos una aprobación de un revisor antes de poder fusionarse. Sea constructivo y respetuoso durante las revisiones de código.
